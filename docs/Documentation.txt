Documentación del módulo: metaobjects/shopify_metaobjects
============================================================

Toolkit modular para gestionar metaobjetos de Shopify vía Admin GraphQL.
Usa execute_admin_graphql de shopify_requests (mismo flujo de tokens que el resto del proyecto).


Clases principales
------------------

1. Metaobject — instancia de un metaobjeto
2. MetaobjectDefinitionSpec — definición de tipo (schema)
3. MetaobjectFieldSpec — campo dentro de una definición
4. MetaobjectClient — cliente de alto nivel para operaciones GraphQL


Metaobject
----------

Representa un metaobjeto individual.

Atributos:
  id (Optional[str])
  type (str)
  handle (str)
  fields (dict[str, Any])
  metafields (dict[str, dict[str, Any]])

Métodos de clase:
  from_shopify_data(data) — parsea respuesta GraphQL
  from_dict(data, *, type, handle_key="handle") — construye desde dict
  from_dataclass(instance, *, type, handle_field="handle") — construye desde dataclass

Métodos de instancia:
  to_shopify_fields() — formato para metaobjectUpsert
  get_field(key) / set_field(key, value)
  get_metafield(key, namespace="custom") / set_metafield(...)
  to_dict()


MetaobjectDefinitionSpec
------------------------

Define el schema de un tipo de metaobjeto.

Atributos:
  type (str)
  name (str)
  description (str)
  fields (list[MetaobjectFieldSpec])

Métodos:
  to_shopify_input() — payload para metaobjectDefinitionCreate
  from_dict(data) — desde respuesta GraphQL o dict manual
  from_dataclass(dc_type, *, type, name, description, handle_field="handle")
      — genera schema desde dataclass; excluye el campo handle


MetaobjectFieldSpec
-------------------

  key, name, type, description, required, validations
  to_shopify_input() — campo individual para create definition


MetaobjectClient
----------------

  __init__(shop: str)

  get_definition(type) -> MetaobjectDefinitionSpec | None
      Obtiene definición existente por type.

  ensure_definition(spec) -> MetaobjectDefinitionSpec
      Idempotente: devuelve existente o crea nueva.

  create_definition(spec) -> MetaobjectDefinitionSpec
      Crea definición; lanza DefinitionError si falla.

  get_by_handle(type, handle) -> Metaobject | None
      Lee instancia por handle.

  upsert(metaobject, *, definition=None, validate=True) -> Metaobject
      Crea o actualiza instancia. Validación opcional contra definition.

  upsert_many(metaobjects, *, definition=None, validate=True) -> dict
      Retorna {"upserted": N, "failed": M}.

  sync(data, *, definition, ensure_definition=True, validate=True) -> Metaobject
      Flujo completo: ensure definition + upsert.
      data puede ser dict, dataclass instance, o Metaobject.


Excepciones
-----------

  MetaobjectError — base
  DefinitionError — operaciones de definición (.error_code, .user_errors)
  UpsertError — operaciones de upsert (.error_code, .user_errors)


Mapeo de tipos Python → Shopify
---------------------------------

  str  → single_line_text_field
  int  → number_integer
  float → number_decimal
  bool → boolean
  dict, list → json

Override: field(metadata={"shopify_type": "multi_line_text_field"})


Ejemplo completo
----------------

  from dataclasses import dataclass
  from metaobjects import MetaobjectClient, MetaobjectDefinitionSpec

  @dataclass
  class FabricSpec:
      handle: str
      fabric_name: str
      stretch_level: int
      is_organic: bool

  client = MetaobjectClient("mi-tienda.myshopify.com")
  definition = MetaobjectDefinitionSpec.from_dataclass(
      FabricSpec, type="fabric", name="Fabric", description="..."
  )
  client.sync(
      FabricSpec("main-cotton", "Classic Cotton", 2, True),
      definition=definition,
  )
